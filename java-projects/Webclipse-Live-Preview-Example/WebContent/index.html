GETTING STARTED WITH LIVE PREVIEW

Live Preview, an exciting time-saver for the modern web developer, is available in MyEclipse and Webclipse.

Preview HTML and CSS changes live in the browser

You know that "save/reload dance" we've been doing for years? The one where you make changes in your editor, hit save, switch to the browser and then refresh to finally see the result? With Live Preview, you don't have to do that dance.

Live Preview opens a live connection to your local browser and pushes HTML and CSS updates as you type! You might already be doing something like this today with browser-based tools, but with Live Preview there is no need to copy and paste the final code back into the editor. Your code runs in the browser, but lives in your editor!

Live Highlight HTML elements and CSS rules

Webclipse makes it easy to see how your changes in HTML and CSS will affect the page. When your cursor is on a CSS rule, Live Preview highlights all affected elements in the browser. Similarly, when editing an HTML file, Live Preview highlights the corresponding HTML elements in the browser.

If you have any modern browser installed, you can try this out yourself. Simply deploy your application to a Java EE application server and open the page. When you deploy, the files are automatically set up to connect back to Live Preview to relay changes. You can turn on Live Preview for one or more servers you are deploying to. Now, place your cursor on the tag above. Notice the blue highlight that appears around the image in the browser. Next, open up the separate CSS file. Try changing the size of the border from 10px to 20px or change the background color from "transparent" to "hotpink". If you have your editor and your browser running side-by-side, you will see your changes instantly reflected in your browser. Cool, right?

Live Preview is available for both HTML and CSS. Plus, if you use the JavaScript debugger along side, you can live update JavaScript as well!
